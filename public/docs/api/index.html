Connect Four — Технічна Документація

Вітаємо у хабі API-документації навчального проєкту Connect Four. Тут ви знайдете усю технічну інформацію про архітектуру та вихідний код гри.

🎯 Про проєкт

Connect Four — це повноцінна веб-версія класичної настільної гри "Чотири в ряд", створена на React + Vite. Головне завдання гравців — по черзі кидати свої фішки у стовпці так, щоб першим зібрати переможну комбінацію фішок одного кольору в ряд (по горизонталі, вертикалі або діагоналі). Гра підтримує гнучкі налаштування та веде підрахунок перемог.

Ключовий функціонал:

  • ⚙️ Можливість налаштувати кількість фішок для перемоги (winLength) та необхідну кількість перемог для виграшу матчу.
  • 🎨 Кастомізація кольорів фішок для кожного гравця.
  • 🍪 Регулювання налаштувань файлів cookie для відповідності GDPR.
  • 💾 Збереження налаштувань гри у локальному сховищі (localStorage).

🛠 Технологічний стек

Проєкт використовує сучасний стек веб-розробки:

  • Основний фреймворк: React (v18)
  • Інструмент збірки: Vite (v5)
  • Управління формами: react-hook-form & Yup
  • Сповіщення про файли cookie: react-cookie-consent
  • Маршрутизація: react-router-dom
  • Документація коду: JSDoc
  • Тестування UI-компонентів: Storybook

🎨 Storybook (Каталог Компонентів)

Ми використовуємо Storybook для розробки та тестування UI-елементів ізольовано від основної бізнес-логіки додатка. Ви можете запустити локальний сервер (npm run storybook) і взаємодіяти з компонентами (Button, GameBoard, Cell, Column, GameOverModal тощо).

📚 API Документація 👈 (Використовуйте боковий Navbar)

Ця сторінка згенерована автоматично завдяки JSDoc. За допомогою бокового меню (Navbar) праворуч ви можете переглядати детальну структуру проєкту:

  1. Компоненти (Components): Як вони влаштовані, які Props приймають і за що відповідають (наприклад, GameBoard, Cell, Modal).
  2. Хуки (Hooks): Спеціальні функції для роботи з реактивним станом гри (наприклад, useGameState).
  3. Глобальні контексти (Contexts): Управління глобальними налаштуваннями гри (GameSettingsContext).
